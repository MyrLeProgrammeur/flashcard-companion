# CLAUDE.md — `flashcard-companion` project

Android app that replaces AnkiDroid as the review client for the decks generated by
the sibling project `flashcard-pipeline`, with an AI explanation button grounded in
the source PDF. See `README.md` for the quick start.

## Context to load before acting

- The stable GUID `sha1(subject\x1ftheme\x1fquestion)` (see the pipeline's
  `db_writer.py`) is the join key — never `notes.id`.

## Rules

- The backend runs **on the phone via Termux**, not on the PC.
- The Infercom key (`INFERCOM_API_KEY`) lives only in `backend/.env` **on the
  phone**, outside any Syncthing folder — never committed, never on the PC.
- The SRS state (`backend/srs_store.db`) is specific to this app, never written into
  the synced `.apkg` files (avoids Syncthing conflicts).
- PDF↔card matching = MVP heuristic (see `source_matcher.py`) — not guaranteed,
  explicit fallback to explanation from the card alone.
- **Never a `Co-Authored-By: Claude` trailer** — nowhere (commits, PR, artifacts).

## Workflow

- UI dev/iteration: `uvicorn main:app --reload` locally, open `localhost:8420`
  in a PC browser — no need for the phone for this.
- Unit tests: `pytest backend/tests/`.
- Phone-side tasks (Termux install, APK, checking Syncthing paths) = dedicated
  session, cable or wireless debugging. The phone is reachable via `adb` from the
  PC when plugged in/paired (already authorized): Claude can build and
  `adb install` the APK directly, no need for Matheo to do it manually.
